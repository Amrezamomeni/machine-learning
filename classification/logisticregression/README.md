## نکات کلیدی تحلیل (Deep Dive)

### الف) تفسیر ضرایب (Coefficients Interpretation)
برخلاف مدل‌های "جعبه سیاه" (مانند Deep Learning یا Random Forest)، رگرسیون لجستیک به ما اجازه می‌دهد بفهمیم **هر ویژگی چه تأثیری بر احتمال کلاس مثبت دارد**. ضرایب مدل، لگاریتم نسبت شانس (Log-Odds) را تغییر می‌دهند.

### ب) انتخاب Solver مناسب
در پروژه از پارامتر `solver` استفاده شده است:
- `liblinear`: مناسب برای داده‌های کوچک و استفاده از تنظیم‌کننده‌های L1/L2.
- `lbfgs`: پیش‌فرض و بهینه برای داده‌های بزرگتر.
- `saga`: برای داده‌های بسیار بزرگ و ElasticNet.

### ج) اهمیت آستانه تصمیم‌گیری (Threshold Tuning)
پیش‌فرض مدل `0.5` است، اما در پروژه‌های واقعی (مانند پزشکی)، ما آستانه را تغییر می‌دهیم تا **Recall** را بهینه‌سازی کنیم (کاهش False Negatives).
